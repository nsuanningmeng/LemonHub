# 🍋 LemonHub 子站 / 代理加盟 · 保姆级教程

> 本文教你**从零开始**把 LemonHub 跑成一个「平台方（主站）+ 多个代理（子站）」的分销体系。
> 分两条主线：**主站站长**（平台方）和**子站站长**（代理）。每一步都告诉你 **要准备什么** + **点哪里**。
> 英文版：[subsite-guide.en.md](./subsite-guide.en.md)

---

## 0. 先搞懂 5 个概念（2 分钟）

| 概念 | 一句话解释 |
|---|---|
| **主站（site_id = 0）** | 平台方自己的站。拥有部署、AI 通道（上游 API Key）、按**批发价**卖额度。第一个注册的账号就是主站超级管理员。 |
| **子站（白标租户）** | 一个独立品牌的站点：自己的**域名**、名称、Logo、公告、首页文案。归属某个**代理**。一套部署 + 一个数据库可以有很多子站。 |
| **代理（子站站长）** | 拥有并自助管理一个子站的人。在自己的域名上收自己的钱、服务自己的用户。 |
| **采购钱包** | 代理向平台**预付**的余额（整数「厘」记账，1 元 = 1000 厘）。用户每充值一次，平台就按批发价从这个钱包扣一笔。**钱包扣光 → 该子站在线充值入口自动消失（降级）**。 |
| **折扣率 DiscountRate** | 代理拿货的折扣，万分比：`10000` = 原价（无差价），`7000` = 七折。**批发成本 = 面值 × 折扣率 ÷ 10000**，差价就是代理利润。 |

### 资金怎么流（例：折扣 7000＝七折）

```
终端用户在「子站」支付 ¥100
        │ （进的是代理自己的易支付商户，¥100 归代理）
        ▼ 易支付异步回调 LemonHub（单数据库事务、幂等）
   用户到账 ¥100 的额度   +   代理采购钱包扣 ¥70
                                └─ 差价 ¥30 = 代理这单的利润
```

> 关键点：**用户的钱进代理的商户，平台只从代理的「采购钱包」按批发价扣货款。** 所以代理必须先把采购钱包充够，否则就会触发「降级」——用户在该子站看不到在线充值。

## 1. 两边各要准备什么（开工前对照清单）

### 🧑‍💼 主站站长 准备清单
- [ ] 一台服务器 + 一个**主域名**（如 `lemonhub.com`），已部署 LemonHub（见 [README](../README.md)）。
- [ ] 数据库（生产建议 MySQL ≥ 5.7.8 或 PostgreSQL ≥ 9.6）+ Redis（多机/缓存推荐）。
- [ ] 已配置好 **AI 通道（渠道）/ 模型 / 分组 / 倍率**（这是你卖的「货」）。
- [ ] 完成 **支付合规确认**（系统设置里的合规条款，付费功能的前置开关）。
- [ ] 能给子站域名做 **DNS 解析 + 反向代理**（让 `子站域名` 指向本部署）。
- [ ] （可选）主站自己的易支付商户，用于主站自营充值。

### 🧑‍🔧 子站站长（代理）准备清单
- [ ] 一个**子站域名**（如 `agent-a.com` 或 `ai.agent-a.com`），愿意 CNAME/解析到平台部署。
- [ ] 一套**自己的易支付（彩虹易支付等）商户**：`商户ID`、`商户密钥`、`支付网关地址`，并支持 `alipay`/`wxpay` 等。
- [ ] 在易支付商户后台，把**回调/通知域名**设置为允许你的子站域名（关键！否则「已付款显示未支付」）。
- [ ] 一个 Logo 图片 URL（可选）、站点名称、公告/页脚文案。
- [ ] 向主站站长预付一笔**采购钱包**余额。

## 2. 主站站长操作（Step-by-Step）

> 全程用**主域名**登录主站超级管理员账号操作。

### 第 1 步 · 基础配置（只做一次）
1. 登录主站后台 → `设置 → 系统设置`：填好 **服务器地址（ServerAddress）** 为你的主域名（如 `https://lemonhub.com`）。
2. `设置` 里完成 **支付合规确认**（勾选合规条款）。未确认时，所有站点的充值/兑换等付费入口都不会出现。
3. `渠道 / 模型 / 分组 / 倍率`：把你要卖的模型、上游渠道、分组倍率配好。子站用户最终用的就是这些通道。

### 第 2 步 · 为代理创建一个「主站账号」
> 子站的「归属代理」必须先是一个**主站的普通用户账号**（不能是管理员/超级管理员）。

1. `用户管理 → 新建用户`，建一个普通用户，例如用户名 `agentA`，设好初始密码。
2. 记下这个用户名，下一步创建子站时要用它做「归属代理」。

> 创建子站时，系统会自动把 `agentA` 提升为「子站管理员」并把它**绑定到新子站**。之后 `agentA` 就只在该子站域名下登录、只能管理自己的子站。

### 第 3 步 · 创建子站
1. 左侧菜单进入 **「子站管理」** 页（主站管理员专属）。
2. 点 **「新建子站」**，在右侧抽屉里填写：

| 字段 | 填什么 | 说明 |
|---|---|---|
| **名称 Name** | 代理的品牌名，如 `A 智能` | 显示在该子站标题/品牌处 |
| **域名 Domains** | 每行一个，如 `agent-a.com` | **至少一个**；一个域名只能绑一个子站 |
| **归属代理 Owner Username** | `agentA` | 必须是**主站的普通用户**（非管理员） |
| **Logo URL** | 图片地址（可选） | 子站 Logo |
| **状态 Status** | `Normal` | `Disabled` = 停用该子站 |
| **折扣率 Discount Rate** | 如 `7000`（七折） | `10000`=原价。批发成本 = 面值 × 它 ÷ 10000 |
| **钱包预警阈值 Wallet Warn Threshold** | 如 `50000`（= ¥50，单位**厘**） | 余额低于它给提醒；`0`=不提醒 |
| **公告 Notice / 页脚 Footer** | 可选 | 公告支持 Markdown |
| **首页 Hero（Badge / Title 1 / Title 2）** | 可选 | 子站首页大标题白标文案，留空用默认 |
| **高级 · Pay Config** | 一般留空 | 代理收款一般由代理自己在子站控制台填；这里是 JSON 高级入口 |

3. 点 **「保存」**。创建成功后，`agentA` 已被提升为子站管理员并绑定到该子站。

> 💡 **单位提醒**：创建抽屉里的「钱包预警阈值」单位是**厘**（1 元 = 1000 厘，¥50 = `50000`）。而后面的「钱包充值」对话框是按**元**输入的，别填错。

### 第 4 步 · 给代理的采购钱包充值
> 代理「拿货」的本钱。没钱 = 该子站在线充值降级消失。

1. 在「子站管理」列表里找到该子站 → 点行操作里的 **「钱包」**。
2. 在 **钱包管理** 抽屉：
   - **充值 Recharge**：填「充值金额（元）」+ 备注 → 提交。这会**增加**代理采购钱包余额（代表代理向你打款拿货）。
   - **手动调整 Manual Adjustment**：填金额（可填**负数**做扣减/纠错）+ **必填**备注。
   - **钱包流水 Wallet Logs**：每一笔变动都有记录，可对账。
3. 充值后，余额立即生效。

### 第 5 步 · 把子站域名指向本部署（DNS + 反代）
1. 让代理把子站域名（`agent-a.com`）解析到你的服务器（A 记录或 CNAME）。
2. 在你的反向代理（Nginx / Caddy / Cloudflare 等）把该域名转发到 LemonHub 容器（默认 `:3000`），并配好 HTTPS 证书。
3. **务必保留并透传 `Host` 头**（LemonHub 靠 `Host` 把请求路由到对应子站）。Nginx 例：`proxy_set_header Host $host;`
4. 打开 `https://agent-a.com`，应当显示该子站的品牌（名称/Logo）。如果显示主站品牌，说明域名没匹配上（检查域名拼写、缓存、Host 透传）。

### 第 6 步 · 日常运维
- **对账**：「子站管理」页有 **「对账 Reconcile」**，核对各子站钱包账。
- **预警**：余额低于阈值时关注，提醒代理及时续费。
- **停用**：把子站状态改 `Disabled` 即下线该子站（不影响其他站）。
- **改折扣/品牌**：编辑子站抽屉随时调整（归属代理在创建后固定，不可改）。

## 3. 子站站长（代理）操作（Step-by-Step）

> 全程用**你自己的子站域名**登录（如 `https://agent-a.com/login`），账号是主站站长给你的 `agentA`。

### 第 1 步 · 登录并进入「子站管理控制台」
1. 浏览器打开 `https://agent-a.com`，用 `agentA` 登录。
2. 进入 **「子站管理控制台 / 代理控制台（Agent Console）」**。它有 4 个标签页：

| 标签 | 作用 |
|---|---|
| **总览与钱包 Overview & Wallet** | 看采购钱包余额、预警阈值、近期流水 |
| **兑换码 Redemption Codes** | 生成 / 作废本站兑换码 |
| **品牌设置 Branding Settings** | 改名称 / Logo / 公告 / 页脚 / 首页文案 |
| **收款设置 Payment Settings** | 配置你自己的易支付商户（**最关键**） |

### 第 2 步 · 配置品牌（Branding Settings）
- 填站点名称、Logo、公告、页脚、首页 Hero 文案 → 保存。终端用户看到的就是你的品牌。

### 第 3 步 · 配置收款（Payment Settings）— 重中之重
> 这一步决定用户的钱进**你的**口袋。需要你先有一套易支付商户。

在 **收款设置** 填：

| 字段 | 填什么 | 例 |
|---|---|---|
| **商户ID Merchant ID** | 你易支付商户的 PID | `1001` |
| **商户密钥 Merchant Key** | 你易支付商户的密钥 | `（保密）` |
| **支付网关地址 Payment Gateway URL** | 你易支付平台的网关地址 | `https://pay.example.com` |
| **支付方式 Payment Methods** | 逗号分隔 | `alipay,wxpay` |

填完点 **保存**。

> ⚠️ **回调域名一定要对**（否则「已付款但显示未支付」）：
> 1. 在你的**易支付商户后台**，把允许的「异步通知 / 回调域名」设置为你的子站域名（`agent-a.com`）。
> 2. 确认子站域名已正确解析 + 反代到 LemonHub（第 2 节第 5 步）。
> LemonHub 对**子站**充值会用「子站自己的域名」回调，并用「子站自己的密钥」验签，所以这两点必须匹配。

### 第 4 步 · 自检一笔充值
1. 用一个普通用户账号在 `https://agent-a.com` 注册/登录。
2. 进充值页，应当能看到在线充值入口（前提：**钱包有余额** + **收款已配置**）。
3. 小额充值走通：支付成功后用户额度到账、你的采购钱包按批发价扣减、流水里能看到。
4. 若看不到在线充值入口 → 多半是**采购钱包余额 ≤ 0** 或**收款未配置完整**（降级）。

### 第 5 步 · 日常运营
- **兑换码**：在「兑换码」页按需生成发给用户。
- **保持钱包充足**：余额会随用户充值消耗，**提前向主站站长续费**，避免降级断充。
- **看流水**：总览页可查钱包流水对账。

## 4. 计费 / 折扣率 算账示例

设面值单价、分组倍率等已在主站配好，最终用户某次充值「面值 ¥M」：

| 折扣率 | 用户付（进代理商户） | 代理采购钱包扣（批发） | 代理毛利 | 用户到账额度 |
|---|---|---|---|---|
| `10000`（原价） | ¥M | ¥M | ¥0 | 面值 ¥M 对应额度 |
| `8000`（八折） | ¥M | ¥M × 0.8 | ¥M × 0.2 | 面值 ¥M 对应额度 |
| `7000`（七折） | ¥M | ¥M × 0.7 | ¥M × 0.3 | 面值 ¥M 对应额度 |
| `5000`（五折） | ¥M | ¥M × 0.5 | ¥M × 0.5 | 面值 ¥M 对应额度 |

> 折扣率**越低**，代理拿货越便宜、利润越高；主站靠走量赚批发差。具体「面值 → 额度」由主站的单价/分组倍率决定。

## 5. 常见问题 / 排错（FAQ）

**Q1：用户付了钱，易支付显示成功，但子站显示未支付？**
- 子站**收款回调域名**没配对：易支付商户后台允许的回调域名 ≠ 子站域名；或子站域名没解析/反代到 LemonHub。
- 子站域名的反代**没透传 `Host`**：LemonHub 收不到正确 Host → 验签/路由异常。
- 偶发：易支付回调网络抖动。LemonHub 的回调端点已豁免 gzip/全局限流并保证返回纯 `success`；若仍偶发，可让主站用「补单」（管理员手动完成订单）兜底。

**Q2：子站看不到「在线充值」入口？**
- 该子站**采购钱包余额 ≤ 0**（降级）→ 主站给钱包充值。
- **收款未配置完整**（商户ID/密钥/网关地址缺一）→ 子站在「收款设置」补全。
- 主站**未完成支付合规确认** → 主站去系统设置确认。

**Q3：访问子站域名却显示主站品牌？**
- 域名没匹配上：检查「子站管理」里域名拼写、是否已保存、DNS 是否生效、反代 `Host` 透传、浏览器/CDN 缓存。

**Q4：代理（agentA）用主域名登录不进自己的控制台？**
- 代理账号在创建子站时已被**绑定到子站**，请用**子站域名**登录（`https://agent-a.com/login`）。

**Q5：能改子站的归属代理吗？**
- 归属代理在创建时固定（涉及角色与 site_id 绑定），不可在编辑中更改。品牌/折扣/状态/收款可随时改。

**Q6：钱包余额单位为什么有时是「厘」有时是「元」？**
- 内部账本用**整数厘**（1 元 = 1000 厘）保证精度不丢。创建抽屉的「钱包预警阈值」按**厘**填；钱包充值对话框按**元**输入、界面按 ¥ 展示。

## 6. 安全 / 合规提示

- **数据隔离**：各子站数据按 `site_id` 隔离；用户名按站唯一；密码/2FA/OAuth 按站隔离。代理看不到其他站数据。
- **代理权限边界**：子站管理员只能管自己的站（品牌/收款/钱包查看/兑换码），改不了域名/折扣/归属（这些归主站）。
- **资金正确性**：钱包为整数厘、原子条件扣减、绝不为负、每次变动写流水；充值结算幂等（重复/并发回调不会重复发放）。
- **合规**：对外提供生成式 AI 服务，请主站方完成备案、内容安全、实名、日志留存、税务、支付与上游授权等义务；代理的收款合规由代理自行承担。

## 7. 快速对照（一页速查）

**主站站长：** 部署 → 配通道/模型/分组 → 合规确认 → 建代理主站账号 → 建子站（域名+归属+折扣+品牌）→ 给钱包充值 → 子站域名 DNS+反代（透传 Host）→ 对账/续费/运维。

**子站站长：** 拿账号 → 子站域名登录 → 进代理控制台 → 配品牌 → 配收款（易支付商户 + 回调域名）→ 自检一笔充值 → 发兑换码/运营 → 余额不足前向主站续费。
